====test====
When I send a request to an operation with Accept header containing text/plain, application/json
And this operation can produce a response with content-type of text/plain
Then I should get back a response with content-type of text/plain
And this response should have 200 status code
====spec====
openapi: 3.0.2
paths:
  /todos:
    get:
      responses:
        200:
          description: Get Todo Items
          content:
            'application/xml':
              example: <project><foo>1</foo><bar>2</bar></project>
            'text/plain':
              example: hello
====server====
mock -p 4010
====command====
curl -i -X GET http://localhost:4010/todos -H "accept: text/plain, application/json"
====expect====
HTTP/1.1 200 OK
content-type: text/plain
content-length: 5
Connection: keep-alive

hello
