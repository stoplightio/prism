====test====
When I send a request to an operation
And the operation has a required NOT exploded form query param
And in the request I sent the param name and object properties with values with invalid types (strings not integers)
Then I should get a 422 response back
====spec====
openapi: '3.0.2'
paths:
  /recipes:
    get:
      parameters:
        - name: pagination
          in: query
          required: true
          style: form
          explode: false
          schema:
            type: object
            properties:
              offset:
                type: integer
                example: 0
              limit:
                type: integer
                example: 25
      responses:
        '200':
          description: OK
====server====
mock -p 4010 ${document}
====command====
curl -i -X GET http://127.0.0.1:4010/recipes\?pagination\=limit,string,offset,string
====expect====
HTTP/1.1 422 Unprocessable Entity
content-type: application/problem+json

{"type":"https://stoplight.io/prism/errors#UNPROCESSABLE_ENTITY","title":"Invalid request","status":422,"detail":"Your request is not valid and no HTTP validation response was found in the spec, so Prism is generating this error for you.","validation":[{"location":["query","pagination","offset"],"severity":"Error","code":"type","message":"Request query parameter pagination.offset must be integer"},{"location":["query","pagination","limit"],"severity":"Error","code":"type","message":"Request query parameter pagination.limit must be integer"}]}
