====test====
Adding a header of "X-API-Key: <<API_KEY>>" for a request to an endpoint,
which security depends on manually selected ApiKeyAuth from multiple security definitions, results in granted access.
====spec====
openapi: 3.0.2
paths:
  /todos:
    get:
      security:
        - ApiKeyAuth: []
      responses:
        200:
          description: OK
          content:
            application/json:
              example: test
components:
  securitySchemes:
    BasicAuth:
      type: http
      scheme: basic
    ApiKeyAuth:
      type: apiKey
      in: header
      name: X-API-Key
security:
  - BasicAuth: []
  - ApiKeyAuth: []
====server====
mock -p 4010 ${document}
====command====
curl -i http://localhost:4010/todos -H "X-API-Key: abc123"
====expect====
HTTP/1.1 200 OK

"test"
