====test====
When prism is started with the --otel-telemetry flag, OpenTelemetry is enabled but
responses are still served normally (the server span wraps the handler transparently).
====spec====
openapi: 3.0.2
paths:
  /todos:
    get:
      responses:
        200:
          description: Get Todo Items
          content:
            'application/json':
              example: hello
====server====
mock -p 4010 --otel-telemetry ${document}
====command====
curl -i -X GET http://localhost:4010/todos -H "accept: application/json"
====expect-loose====
HTTP/1.1 200 OK
content-type: application/json

"hello"
